# V1-1-22

## box member 승인/거절

### BoxStatus 필드 추가

`user/{email}` 문서에 `status` 필드가 추가되었습니다. 기존에는 `boxName`의 `?` 접두사만으로 상태를 구분했으나, `PENDING`과 `REJECTED`를 명시적으로 구별할 수 없었습니다.

**BoxStatus 값:**
- `NONE` — 소속 박스 없음
- `PENDING` — 가입 신청 후 승인 대기 중 (`?boxName` 유지)
- `APPROVED` — 승인 완료 (clean `boxName`)
- `REJECTED` — 가입 거절됨 (`?boxName` 유지)

### 거절(REJECTED) 동작 변경

기존에는 거절 시 `boxName`을 빈 문자열로 초기화했으나, 이제 `?boxName`을 유지하고 `status: REJECTED`만 기록합니다.

- 변경 전: `deleteApplication` + `updateUserBoxName(email, '')`
- 변경 후: `deleteApplication` + `updateUserStatus(email, 'REJECTED')`

`?boxName` 유지로 사용자가 어느 박스에 거절됐는지 추적 가능합니다.

### 승인(APPROVED) 동작

`removeApplication` → `updateUserBoxInfo(email, boxName, 'APPROVED')` — boxName(`?` 제거)과 status를 한 번에 업데이트합니다.

### 기존 사용자 호환

`status` 필드가 없는 기존 사용자는 `boxName`의 `?` 접두사로 상태를 추론합니다 (`deriveBoxStatus` 함수).
