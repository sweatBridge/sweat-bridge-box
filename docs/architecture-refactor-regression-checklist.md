# 아키텍처 리팩터링 회귀 테스트 체크리스트

## 목적

이 체크리스트는 `feature/V1-1-7`의 리팩터링 후에도 사용자 관점 동작이 유지되는지 수동으로 점검하기 위한 문서다.
자동 테스트가 커버하지 못하는 UI 연결과 실제 흐름 위주로 확인한다.

---

## 4. 수동 회귀 테스트

## 회원 관리

- 회원 목록 진입 시 로딩 후 목록이 정상 표시된다.
- 회원 상태 배지(신규/주의/활성/비활성)가 기존과 동일하게 보인다.
- 검색어 입력 시 실명 기준 검색 결과가 동일하다.
- 만료 탭과 현재 탭 분리가 기존과 동일하게 동작한다.
- 회원 상세/관리 모달 진입이 정상 동작한다.
- 회원 삭제 후 목록과 카운트가 정상 갱신된다.

## 회원권 관리

- 회원권 추가 시 기간 겹침 검증이 기존과 동일하게 동작한다.
- 기간권 추가 후 만료일과 상태 배지가 정상 반영된다.
- 횟수권 추가 후 남은 횟수와 상태 배지가 정상 반영된다.
- 홀딩 등록 시 종료일이 기대한 일수만큼 연장된다.
- 미래 홀딩 해제 시 종료일과 후속 회원권 시작일이 원복된다.
- 현재 홀딩 해제 시 종료일 단축과 조정 이력이 정상 반영된다.
- 환불 처리 후 회원 상태와 매출 반영이 동일하다.
- 전체 연장 실행 후 활성 회원권만 연장된다.

## 수업 관리

- 월간 캘린더 진입 시 수업 목록이 정상 렌더링된다.
- 수업 생성 시 시간, 코치, 정원이 기존과 동일하게 저장된다.
- 수업 수정 시 정원/예약자 정보가 유지된다.
- 수업 삭제 후 캘린더에서 즉시 제거된다.
- 예약 회원 보기 모달이 정상 동작한다.

## 락커 관리

- 락커 목록 진입 시 번호순으로 정상 표시된다.
- 빈 락커 배정 시 회원 정보와 기간이 정상 반영된다.
- 배정 중인 락커에 재배정 시 차단 메시지가 기존과 동일하다.
- 락커 해지 후 상태가 사용 가능으로 바뀐다.
- 락커 삭제 시 삭제 상태가 정상 반영된다.
- 전체 연장 실행 시 만료되지 않은 사용 중 락커만 연장된다.
- 락커 히스토리 모달에서 과거 이력이 정상 표시된다.

## 매출 관리

- 월별 매출 총액이 기존과 동일하다.
- 회원권 매출과 기타 매출 분리가 동일하다.
- 현금/카드 집계가 동일하다.
- 환불 금액 반영 후 총매출 계산이 동일하다.
- 거래 내역 모달에서 일자별 거래가 정상 표시된다.

## 공통 확인

- 브라우저 콘솔에 새 에러가 발생하지 않는다.
- Firestore에 저장되는 문서 경로와 주요 필드가 기존과 동일하다.
- 레거시 회원 데이터(`birth`만 있는 회원)도 화면에 정상 표시된다.
- 리팩터링 전과 비교했을 때 주요 화면의 로딩/성공/실패 UX가 변하지 않는다.
